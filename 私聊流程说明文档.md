# 私聊完整流程说明文档

## 一、概述

本系统的私聊功能基于 **WebSocket 实时通信** + **REST API 数据同步** 的混合架构实现。整个流程包括三个核心环节：

1. **消息发送**：从发送者输入到保存到数据库
2. **消息接收**：接收者通过 WebSocket 实时收到消息
3. **消息已读**：标记消息已读状态并同步

connect(userId: number): Promise<void> {

# backend/app/websocket/handlers.py:224-281
async def handle_private_message(db: Session, user_id: int, message: dict):
    receiver_id = message.get("receiver_id")
    content = message.get("content")
    message_type = message.get("message_type", "text")
    
    # 1. 参数校验
    if not receiver_id or not content:
        await manager.send_personal_message({"type": "error", "message": "缺少必要参数"}, user_id)
        return
    
    # 2. 创建消息并保存到数据库
    private_message = create_private_message(db, user_id, receiver_id, content, message_type)
    
    # 3. 刷新对象以加载关联关系（sender, receiver）
    db.refresh(private_message)
    
    # 4. 构建响应消息（包含完整信息）
    response_message = {
        "type": "private_message",
        "id": private_message.id,
        "sender_id": user_id,
        "receiver_id": receiver_id,
        "content": content,
        "message_type": message_type,
        "is_read": False,
        "created_at": private_message.created_at.isoformat(),
        "sender_username": private_message.sender.username if private_message.sender else None,
        "receiver_username": private_message.receiver.username if private_message.receiver else None
    }
    
    # 5. 发送给接收者
    await manager.send_personal_message(response_message, receiver_id)
    
    # 6. 回发给发送者（确认发送成功）
    await manager.send_personal_message(response_message, user_id)
```

# backend/app/services/chat_service.py:482-530
def create_private_message(
    db: Session,
    sender_id: int,
    receiver_id: int,
    content: str,
    message_type: str = "text"
) -> PrivateMessage:
    
    # 1. 检查接收者是否存在
    receiver = db.query(User).filter(User.id == receiver_id).first()
    if not receiver:
        raise HTTPException(status_code=404, detail="接收者不存在")
    
    # 2. 检查是否给自己发消息
    if sender_id == receiver_id:
        raise HTTPException(status_code=400, detail="不能给自己发送私聊消息")
    
    # 3. 创建消息对象
    message = PrivateMessage(
        sender_id=sender_id,
        receiver_id=receiver_id,
        content=content,
        message_type=message_type
    )
    
    # 4. 保存到数据库
    db.add(message)
    db.commit()
    db.refresh(message)
    
    return message
```

def mark_message_read(db: Session, message_id: int, user_id: int) -> bool:
    message = db.query(PrivateMessage).filter(PrivateMessage.id == message_id).first()
    
    if not message:
        raise HTTPException(status_code=404, detail="消息不存在")
    
    # 只有接收者才能标记已读
    if message.receiver_id != user_id:
        raise HTTPException(status_code=403, detail="无权标记此消息为已读")
    
    if not message.is_read:
        message.is_read = True
        message.read_at = func.now()  # 记录阅读时间
        db.commit()
    
    return True

## 四、数据流转总结

### 4.1 发送消息时的数据流转

```
发送者前端
  ↓ (WebSocket)
  ├─ 消息格式: {type: "private_message", receiver_id, content}
  ↓
后端 WebSocket 处理器 (handlers.py)
  ↓
  ├─ 验证参数
  ├─ 调用 create_private_message() 保存到 DB
  │   └─ private_messages 表: sender_id, receiver_id, content, is_read=false
  ↓
连接管理器 (connection_manager.py)
  ↓
  ├─ 发送给接收者: send_personal_message(response, receiver_id)
  └─ 回发给发送者: send_personal_message(response, user_id)
```

### 4.2 接收消息时的数据流转

```
接收者前端 WebSocket 接收
  ↓
WebSocketService.handleMessage()
  ↓
PrivateChatPage 消息处理器
  ↓
  ├─ 验证消息是否属于当前对话
  ├─ 添加到 messages 状态（渲染到界面）
  └─ 调用 markMessageRead() 标记已读
       ↓ (REST API PUT)
       └─ 后端更新 DB: is_read=true, read_at=now()
```

## 五、关键技术点

### 5.1 WebSocket vs REST API 的分工

| 功能 | 技术方案 | 说明 |
|------|---------|------|
| 消息发送 | WebSocket | 实时性要求高，双向通信 |
| 消息接收推送 | WebSocket | 服务器主动推送 |
| 历史消息加载 | REST API (GET) | 分页拉取，适合初始加载 |
| 标记已读 | REST API (PUT) | 状态更新，单向操作 |
| 会话列表 | REST API (GET) | 定期刷新或进入页面时加载 |

### 5.2 离线消息处理

- **发送时**：无论接收者是否在线，消息都会保存到数据库
- **上线后**：接收者通过 GET `/api/chat/private/{user_id}/messages` 拉取历史消息
- **未读计数**：GET `/api/chat/private/unread-count` 获取未读消息数

### 5.3 心跳检测与重连机制

- **前端**：每 30 秒发送 `ping` 消息检测连接
- **自动重连**：连接断开后最多尝试重连 5 次，每次间隔 3 秒
