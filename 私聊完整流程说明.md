## 分步骤详细说明

### 阶段一：WebSocket连接建立

**触发时机**：用户登录成功后自动建立

**前后端配合流程**：

| 步骤 | 前端 (WebSocketContext.tsx) | 后端 (handlers.py) | 数据流转 |
|------|----------------------------|-------------------|----------|
| 1 | `useEffect` 检测到 `isAuthenticated && user`，调用 `wsService.connect(user.id)` | - | 前端发起连接 |
| 2 | `wsService.connect()` 构建 URL: `WS_URL/ws/{userId}?token={token}` | - | 携带用户ID和JWT Token |
| 3 | - | `main.py:158` 路由 `/ws/{user_id}` 接收连接 | HTTP → WebSocket 升级 |
| 4 | - | `websocket_endpoint()` 先调用 `websocket.accept()` | 建立 TCP 连接 |
| 5 | - | `verify_websocket_token(token)` 验证 JWT | 解码 Token 获取用户ID |
| 6 | - | 验证通过后 `manager.active_connections[user_id] = websocket` | 内存中保存连接映射 |
| 7 | - | 发送 `{type: 'connection', status: 'connected'}` | 前端收到连接成功确认 |

**关键代码位置**：
- 前端：`frontend/src/contexts/WebSocketContext.tsx:38-63`
- 后端：`backend/app/websocket/handlers.py:63-96`

## 关键数据结构

### PrivateMessage 模型（数据库）

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| id | Integer | 主键 | 自增 |
| sender_id | Integer | 发送者用户ID（外键） | - |
| receiver_id | Integer | 接收者用户ID（外键） | - |
| content | Text | 消息内容 | - |
| message_type | String(20) | 消息类型（text/image/file） | 'text' |
| is_read | Boolean 是否已读 | FALSE |
| read_at | DateTime | 读取时间 | NULL |
| created_at | DateTime | 创建时间 | NOW() |

## 前后端配合总结

### 通信方式

| 场景 | 通信方式 | 说明 |
|------|----------|------|
| 发送消息 | WebSocket | 实时性要求高，双向通信 |
| 接收消息 | WebSocket | 服务端主动推送 |
| 历史消息 | REST HTTP GET | 一次性拉取，分页 |
| 标记已读 | REST HTTP PUT | 幂等操作 |
| 会话列表 | REST HTTP GET | 进入页面时拉取 |

### 状态管理

| 状态 | 存储位置 | 同步方式 |
|------|----------|----------|
| WebSocket连接 | 内存 (ConnectionManager) | 心跳检测 + 自动重连 |
| 消息数据 | MySQL 数据库 | 持久化存储 |
| 在线状态 | 内存 (active_connections) | 连接/断开时更新 |
| 前端消息列表 | React useState | WebSocket 实时更新 + API 兜底 |

### 容错机制

1. **WebSocket 自动重连**：`websocket.ts:213-233`，最多5次重试
2. **心跳检测**：每30秒发送 ping，保持连接活跃
3. **连接状态监控**：前端每秒检查连接状态并更新UI
